# Bot API Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Discord-style Bot API to the Together server: static bot tokens, bot user accounts, REST endpoints for bot management, WebSocket gateway support for bots, and per-bot rate limiting.

**Architecture:** Bots are stored as regular `users` rows with `is_bot = true`, authenticated via a static random token (hashed in a `bots` table) using `Authorization: Bot <token>` headers. The existing `AuthUser` extractor is extended to accept both `Bearer <jwt>` and `Bot <token>` auth — all existing REST handlers and the WebSocket gateway work for bots with no handler changes needed.

**Tech Stack:** Rust/Axum, SQLx/PostgreSQL, `governor` crate (already imported) for per-bot rate limiting via `DefaultKeyedRateLimiter<Uuid>`, `rand` for token generation.

```rust
use sha2::{Digest, Sha256};

// ============================================================================
// Token Generation
// ============================================================================

/// Generate a cryptographically random 32-byte bot token, hex-encoded.
/// The plaintext is shown to the user exactly once at creation time.
pub fn generate_bot_token() -> String {
    use std::time::{SystemTime, UNIX_EPOCH};
    // Use OS random bytes via rand (already available via uuid crate internals,
    // but we call rand directly for clarity).
    let mut bytes = [0u8; 32];
    // rand is available because uuid pulls it in, but to be explicit:
    getrandom::getrandom(&mut bytes).expect("Failed to generate random bytes");
    hex::encode(bytes)
}

/// Hash a plaintext bot token with SHA-256 for storage.
/// Same approach as refresh tokens: deterministic for lookup, not bcrypt.
pub fn hash_bot_token(token: &str) -> String {
    let mut hasher = Sha256::new();
    hasher.update(token.as_bytes());
    format!("{:x}", hasher.finalize())
}
```

**Note on `getrandom` and `hex`**: these crates may not be in Cargo.toml yet. Check first:

- `getrandom` is a dependency of `uuid` and `rand` — it's transitively available but not directly importable without adding it explicitly.
- Simpler: use `uuid::Uuid::new_v4()` which is already seeded with OS entropy, then SHA-256 its bytes.

Replace the `generate_bot_token` implementation with:

```rust
pub fn generate_bot_token() -> String {
    // Use two UUIDs concatenated for 32 bytes of OS entropy
    let a = uuid::Uuid::new_v4();
    let b = uuid::Uuid::new_v4();
    let mut hasher = Sha256::new();
    hasher.update(a.as_bytes());
    hasher.update(b.as_bytes());
    format!("{:x}", hasher.finalize())
}
```

This avoids adding new dependencies while remaining cryptographically random.

```rust
#[async_trait]
impl FromRequestParts<AppState> for AuthUser {
    type Rejection = AuthRejection;

    async fn from_request_parts(
        parts: &mut Parts,
        state: &AppState,
    ) -> Result<Self, Self::Rejection> {
        // Extract the raw Authorization header value
        let auth_value = parts
            .headers
            .get(axum::http::header::AUTHORIZATION)
            .and_then(|v| v.to_str().ok())
            .ok_or_else(|| auth_error("Missing or invalid Authorization header"))?;

        if let Some(jwt) = auth_value.strip_prefix("Bearer ") {
            // ── JWT path (existing users) ──────────────────────────────────
            let claims = validate_token(jwt, &state.jwt_secret)
                .map_err(|_| auth_error("Invalid or expired token"))?;

            if claims.token_type != TokenType::Access {
                return Err(auth_error("Invalid token type"));
            }

            let user_id = claims
                .user_id()
                .map_err(|_| auth_error("Invalid token subject"))?;

            Ok(AuthUser { user_id, username: claims.username, is_bot: false })

        } else if let Some(raw_token) = auth_value.strip_prefix("Bot ") {
            // ── Bot static token path ──────────────────────────────────────
            authenticate_bot_token(raw_token, state).await

        } else {
            Err(auth_error("Missing or invalid Authorization header"))
        }
    }
}

/// Look up a bot by its plaintext token, verify it is not revoked,
/// and return an AuthUser backed by the bot's user account.
async fn authenticate_bot_token(
    raw_token: &str,
    state: &AppState,
) -> Result<AuthUser, AuthRejection> {
    use crate::bot_auth::hash_bot_token;

    let token_hash = hash_bot_token(raw_token);

    #[derive(sqlx::FromRow)]
    struct BotRow {
        user_id: uuid::Uuid,
        username: String,
        revoked_at: Option<chrono::DateTime<chrono::Utc>>,
    }

    let row = sqlx::query_as::<_, BotRow>(
        "SELECT b.user_id, u.username, b.revoked_at
         FROM bots b
         JOIN users u ON u.id = b.user_id
         WHERE b.token_hash = $1",
    )
    .bind(&token_hash)
    .fetch_optional(&state.pool)
    .await
    .map_err(|e| {
        tracing::warn!(error = ?e, "DB error during bot token lookup");
        auth_error("Internal authentication error")
    })?;

    let row = row.ok_or_else(|| auth_error("Invalid bot token"))?;

    if row.revoked_at.is_some() {
        return Err(auth_error("Bot token has been revoked"));
    }

    // Apply per-bot rate limiting BEFORE returning AuthUser.
    // This is the cheapest place to enforce limits without handler changes.
    if let Err(_) = state.bot_rate_limiter.check_key(&row.user_id) {
        return Err((
            axum::http::StatusCode::TOO_MANY_REQUESTS,
            axum::Json(serde_json::json!({
                "error": "Bot rate limit exceeded. Max 50 requests/second."
            })),
        ));
    }

    Ok(AuthUser {
        user_id: row.user_id,
        username: row.username,
        is_bot: true,
    })
}
```

```rust
use axum::{
    extract::{Path, State},
    http::StatusCode,
    Json,
};
use serde_json::{json, Value};
use uuid::Uuid;

use crate::{
    auth::AuthUser,
    bot_auth::{generate_bot_token, hash_bot_token},
    error::{AppError, AppResult},
    models::{BotCreatedResponse, BotDto, CreateBotDto},
    state::AppState,
};

// ============================================================================
// POST /bots — Register a new bot
// ============================================================================

/// Create a new bot user and return the plaintext token (shown once only).
///
/// Creates a user row with `is_bot = true`, then inserts a bots row with the
/// hashed token. The plaintext token is returned in the response and never
/// stored — if lost, use POST /bots/:id/token/regenerate.
pub async fn create_bot(
    auth: AuthUser,
    State(state): State<AppState>,
    Json(payload): Json<CreateBotDto>,
) -> AppResult<(StatusCode, Json<BotCreatedResponse>)> {
    // Bots cannot create other bots.
    if auth.is_bot() {
        return Err(AppError::Forbidden("Bots cannot create other bots".into()));
    }

    // Validate name
    let name = payload.name.trim().to_string();
    if name.is_empty() || name.len() > 64 {
        return Err(AppError::Validation("Bot name must be 1–64 characters".into()));
    }

    // Generate a unique username for the bot user account.
    // Format: "{name}-bot-{short_uuid}" to avoid collisions.
    let short_id = Uuid::new_v4().to_string()[..8].to_string();
    let bot_username = format!("{}-bot-{}", slug_name(&name), short_id);

    // Placeholder password hash (bots never log in with password).
    // We store a known-invalid hash so the field is not empty.
    let placeholder_hash = "$2b$12$PLACEHOLDER_BOT_ACCOUNT_NO_PASSWORD_xxxxxxxxxxxxxxxxxxxxxx";

    // Create the bot's user account.
    let bot_user_id: Uuid = sqlx::query_scalar(
        "INSERT INTO users (username, password_hash, status, is_bot)
         VALUES ($1, $2, 'online', TRUE)
         RETURNING id",
    )
    .bind(&bot_username)
    .bind(placeholder_hash)
    .fetch_one(&state.pool)
    .await
    .map_err(|e| {
        tracing::error!(error = ?e, "Failed to create bot user account");
        AppError::Internal
    })?;

    // Generate and hash the bot token.
    let raw_token = generate_bot_token();
    let token_hash = hash_bot_token(&raw_token);

    // Insert into bots table.
    let bot_id: Uuid = sqlx::query_scalar(
        "INSERT INTO bots (user_id, name, description, token_hash, created_by)
         VALUES ($1, $2, $3, $4, $5)
         RETURNING id",
    )
    .bind(bot_user_id)
    .bind(&name)
    .bind(payload.description.as_deref())
    .bind(&token_hash)
    .bind(auth.user_id())
    .fetch_one(&state.pool)
    .await
    .map_err(|e| {
        tracing::error!(error = ?e, "Failed to insert bot record");
        AppError::Internal
    })?;

    let bot = sqlx::query_as::<_, crate::models::Bot>(
        "SELECT id, user_id, name, description, token_hash, created_by, revoked_at, created_at
         FROM bots WHERE id = $1",
    )
    .bind(bot_id)
    .fetch_one(&state.pool)
    .await
    .map_err(|_| AppError::Internal)?;

    tracing::info!(
        bot_id = %bot_id,
        created_by = %auth.user_id(),
        "Bot registered"
    );

    Ok((
        StatusCode::CREATED,
        Json(BotCreatedResponse {
            bot: bot.into(),
            token: raw_token,
        }),
    ))
}

// ============================================================================
// GET /bots — List bots created by the calling user
// ============================================================================

pub async fn list_bots(
    auth: AuthUser,
    State(state): State<AppState>,
) -> AppResult<Json<Value>> {
    if auth.is_bot() {
        return Err(AppError::Forbidden("Bots cannot list bots".into()));
    }

    let bots: Vec<BotDto> = sqlx::query_as::<_, crate::models::Bot>(
        "SELECT id, user_id, name, description, token_hash, created_by, revoked_at, created_at
         FROM bots
         WHERE created_by = $1
         ORDER BY created_at ASC",
    )
    .bind(auth.user_id())
    .fetch_all(&state.pool)
    .await
    .map_err(|e| {
        tracing::error!(error = ?e, "Failed to list bots");
        AppError::Internal
    })?
    .into_iter()
    .map(BotDto::from)
    .collect();

    Ok(Json(json!({ "bots": bots })))
}

// ============================================================================
// GET /bots/:id — Get a single bot
// ============================================================================

pub async fn get_bot(
    auth: AuthUser,
    State(state): State<AppState>,
    Path(bot_id): Path<Uuid>,
) -> AppResult<Json<BotDto>> {
    if auth.is_bot() {
        return Err(AppError::Forbidden("Bots cannot access bot management endpoints".into()));
    }

    let bot = sqlx::query_as::<_, crate::models::Bot>(
        "SELECT id, user_id, name, description, token_hash, created_by, revoked_at, created_at
         FROM bots WHERE id = $1 AND created_by = $2",
    )
    .bind(bot_id)
    .bind(auth.user_id())
    .fetch_optional(&state.pool)
    .await
    .map_err(|_| AppError::Internal)?
    .ok_or_else(|| AppError::NotFound("Bot not found".into()))?;

    Ok(Json(bot.into()))
}

// ============================================================================
// DELETE /bots/:id — Revoke a bot
// ============================================================================

pub async fn revoke_bot(
    auth: AuthUser,
    State(state): State<AppState>,
    Path(bot_id): Path<Uuid>,
) -> AppResult<StatusCode> {
    if auth.is_bot() {
        return Err(AppError::Forbidden("Bots cannot revoke bots".into()));
    }

    let rows_affected = sqlx::query(
        "UPDATE bots SET revoked_at = NOW()
         WHERE id = $1 AND created_by = $2 AND revoked_at IS NULL",
    )
    .bind(bot_id)
    .bind(auth.user_id())
    .execute(&state.pool)
    .await
    .map_err(|_| AppError::Internal)?
    .rows_affected();

    if rows_affected == 0 {
        return Err(AppError::NotFound(
            "Bot not found or already revoked".into(),
        ));
    }

    tracing::info!(bot_id = %bot_id, revoked_by = %auth.user_id(), "Bot revoked");
    Ok(StatusCode::NO_CONTENT)
}

// ============================================================================
// POST /bots/:id/token/regenerate — Regenerate bot token
// ============================================================================

pub async fn regenerate_bot_token(
    auth: AuthUser,
    State(state): State<AppState>,
    Path(bot_id): Path<Uuid>,
) -> AppResult<Json<BotCreatedResponse>> {
    if auth.is_bot() {
        return Err(AppError::Forbidden("Bots cannot regenerate tokens".into()));
    }

    // Verify ownership and not revoked.
    let bot = sqlx::query_as::<_, crate::models::Bot>(
        "SELECT id, user_id, name, description, token_hash, created_by, revoked_at, created_at
         FROM bots WHERE id = $1 AND created_by = $2",
    )
    .bind(bot_id)
    .bind(auth.user_id())
    .fetch_optional(&state.pool)
    .await
    .map_err(|_| AppError::Internal)?
    .ok_or_else(|| AppError::NotFound("Bot not found".into()))?;

    if bot.revoked_at.is_some() {
        return Err(AppError::Validation("Cannot regenerate token for a revoked bot. Create a new bot instead.".into()));
    }

    let raw_token = generate_bot_token();
    let token_hash = hash_bot_token(&raw_token);

    let updated_bot = sqlx::query_as::<_, crate::models::Bot>(
        "UPDATE bots SET token_hash = $1
         WHERE id = $2
         RETURNING id, user_id, name, description, token_hash, created_by, revoked_at, created_at",
    )
    .bind(&token_hash)
    .bind(bot_id)
    .fetch_one(&state.pool)
    .await
    .map_err(|_| AppError::Internal)?;

    tracing::info!(bot_id = %bot_id, "Bot token regenerated");

    Ok(Json(BotCreatedResponse {
        bot: updated_bot.into(),
        token: raw_token,
    }))
}

            }

}
```

```rust
pub async fn websocket_handler(
    ws: WebSocketUpgrade,
    Query(params): Query<WsParams>,
    State(state): State<AppState>,
) -> Response {
    let user_id = if let Some(jwt) = params.token {
        // ── Human user: JWT auth ──────────────────────────────────────────
        let claims = match validate_token(&jwt, &state.jwt_secret) {
            Ok(c) => c,
            Err(_) => return (StatusCode::UNAUTHORIZED, "Invalid or expired token").into_response(),
        };
        if claims.token_type != TokenType::Access {
            return (StatusCode::UNAUTHORIZED, "Access token required").into_response();
        }
        match claims.user_id() {
            Ok(id) => id,
            Err(_) => return (StatusCode::UNAUTHORIZED, "Invalid token subject").into_response(),
        }

    } else if let Some(raw_token) = params.bot_token {
        // ── Bot: static token auth ────────────────────────────────────────
        use crate::bot_auth::hash_bot_token;
        let token_hash = hash_bot_token(&raw_token);

        #[derive(sqlx::FromRow)]
        struct BotLookup { user_id: uuid::Uuid, revoked_at: Option<chrono::DateTime<chrono::Utc>> }

        let row = sqlx::query_as::<_, BotLookup>(
            "SELECT user_id, revoked_at FROM bots WHERE token_hash = $1"
        )
        .bind(&token_hash)
        .fetch_optional(&state.pool)
        .await;

        match row {
            Ok(Some(b)) if b.revoked_at.is_none() => b.user_id,
            Ok(Some(_)) => return (StatusCode::UNAUTHORIZED, "Bot token revoked").into_response(),
            Ok(None)    => return (StatusCode::UNAUTHORIZED, "Invalid bot token").into_response(),
            Err(e) => {
                tracing::warn!(error = ?e, "DB error during bot WS auth");
                return (StatusCode::INTERNAL_SERVER_ERROR, "Auth error").into_response();
            }
        }

    } else {
        return (StatusCode::UNAUTHORIZED, "token or bot_token required").into_response();
    };

    ws.on_upgrade(move |socket| handle_socket(socket, user_id, state))
}
```

```markdown
# Together Bot API

Bots are automated accounts that can send messages, receive real-time events, and interact with Together servers programmatically.

## Authentication

Bots authenticate using a static token with the `Bot` prefix in the Authorization header:
```

Authorization: Bot <your_bot_token>

````

Unlike user JWT tokens (which expire after 15 minutes), bot tokens are long-lived static tokens. Keep your bot token secret — if compromised, regenerate it immediately via the API.

**Do not use `Bearer <jwt>` for bot authentication.** Bot tokens use the `Bot` prefix only.

## Bot Management REST API

All bot management endpoints require user authentication (`Authorization: Bearer <user_jwt>`). Bots cannot manage other bots.

### Register a Bot

**POST** `/bots`

Creates a new bot user account and returns the bot's plaintext token. **The token is shown exactly once.** Store it securely.

**Request body:**
```json
{
  "name": "MyBot",
  "description": "Optional description up to 512 characters"
}
````

**Response:** `201 Created`

```json
{
  "bot": {
    "id": "uuid",
    "user_id": "uuid",
    "name": "MyBot",
    "description": "Optional description",
    "created_by": "uuid",
    "revoked_at": null,
    "created_at": "2026-03-12T00:00:00Z"
  },
  "token": "64-character-hex-token-shown-once-only"
}
```

**Constraints:** `name` must be 1–64 characters.

---

### List Bots

**GET** `/bots`

Returns all bots created by the authenticated user.

**Response:** `200 OK`

```json
{
  "bots": [
    {
      "id": "uuid",
      "user_id": "uuid",
      "name": "MyBot",
      "description": null,
      "created_by": "uuid",
      "revoked_at": null,
      "created_at": "2026-03-12T00:00:00Z"
    }
  ]
}
```

---

### Get a Bot

**GET** `/bots/:id`

Returns a single bot. Only the bot's creator can access it.

**Response:** `200 OK` — same shape as a single bot object above.

**Errors:**

- `404 Not Found` — bot does not exist or belongs to another user

---

### Revoke a Bot

**DELETE** `/bots/:id`

Permanently revokes the bot's token. The bot can no longer authenticate. This action is irreversible — create a new bot if you need a replacement.

**Response:** `204 No Content`

**Errors:**

- `404 Not Found` — bot not found, already revoked, or belongs to another user

---

### Regenerate Bot Token

**POST** `/bots/:id/token/regenerate`

Issues a new token for the bot, immediately invalidating the old one. The new token is shown once.

**Response:** `200 OK` — same shape as POST /bots response (bot + token).

**Errors:**

- `404 Not Found` — bot not found
- `422 Unprocessable Entity` — bot is revoked (create a new bot instead)

## Using the Bot Token for REST API Calls

Once registered, bots can call all standard API endpoints using `Authorization: Bot <token>`:

```bash
# Send a message as a bot
curl -X POST https://your-server.com/channels/{channel_id}/messages \
  -H "Authorization: Bot YOUR_BOT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"content": "Hello from my bot!"}'

# Get current bot user info
curl https://your-server.com/users/@me \
  -H "Authorization: Bot YOUR_BOT_TOKEN"

# List channels in a server
curl https://your-server.com/servers/{server_id}/channels \
  -H "Authorization: Bot YOUR_BOT_TOKEN"
```

Bots must be members of a server to send messages in it. Have a server admin add the bot user to their server using the standard member join flow.

---

## WebSocket Gateway

Bots connect to the gateway using `bot_token` as a query parameter instead of `token`:

```
wss://your-server.com/ws?bot_token=YOUR_BOT_TOKEN
```

### Connection Flow

1. Connect to `/ws?bot_token=<token>`
2. Server authenticates the token and sends a `READY` event
3. Send periodic `HEARTBEAT` messages to keep the connection alive
4. Receive real-time events for servers the bot belongs to

### READY Event

After connecting, the server sends:

```json
{
  "op": "DISPATCH",
  "t": "READY",
  "d": {
    "user": { "id": "...", "username": "mybot-bot-abc12345", "is_bot": true, ... },
    "servers": [...],
    "dm_channels": [],
    "unread_counts": [],
    "mention_counts": []
  }
}
```

### Sending Messages (Gateway)

Bots can also send messages via REST (recommended) rather than through the WebSocket.

### Receiving Events

Bots receive all standard gateway events for servers they belong to:

- `MESSAGE_CREATE` — new message posted
- `MESSAGE_UPDATE` — message edited
- `MESSAGE_DELETE` — message deleted
- `PRESENCE_UPDATE` — user went online/offline
- `REACTION_ADD` / `REACTION_REMOVE` — reactions on messages
- `VOICE_STATE_UPDATE` — users joining/leaving voice channels

### Heartbeat

Bots must send a heartbeat every 30 seconds to stay connected:

```json
{ "op": "HEARTBEAT" }
```

Server responds with:

```json
{ "op": "HEARTBEAT_ACK" }
```

## Rate Limits

Bots are subject to a rate limit of **50 requests/second** per bot token. Exceeding this limit returns:

```json
HTTP 429 Too Many Requests
{ "error": "Bot rate limit exceeded. Max 50 requests/second." }
```

Implement exponential backoff when you receive 429 responses.

The global per-IP rate limit (10 req/s in production) also applies to bot HTTP requests if connecting from a single IP. The per-bot limit runs independently.

---

## Security Best Practices

- **Never commit bot tokens to version control**
- Store tokens in environment variables or a secrets manager
- Regenerate tokens immediately if you suspect compromise
- Bots cannot create or manage other bots — only human users can
- Revoked bots cannot reconnect via WebSocket or make REST calls

- **Token shown once**: Like Discord's bot tokens and API keys in general — if lost, regenerate.
